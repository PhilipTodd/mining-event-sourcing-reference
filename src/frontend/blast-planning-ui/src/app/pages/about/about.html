About

Reference implementation for an Azure-native event-sourced system.

flag Project Purpose Architecture reference

This application demonstrates a production-style event-sourced workflow using Azure PaaS services, CQRS, asynchronous projections, and modern delivery practices.

hub Architecture Summary Command, event, projection, query
  1. Commands are submitted through the API.
  2. Domain events are persisted to Cosmos DB.
  3. Events are published to Azure Service Bus.
  4. A projection worker updates Azure SQL read models.
  5. The UI queries the read model through the API.
